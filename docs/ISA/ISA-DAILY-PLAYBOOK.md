# ISA Daily Playbook — Hour-by-Hour Day Shape

**Audience:** Kareem.
**Purpose:** What every working day looks like, hour by hour, broken out by the actual schedule (Mon 12-6, Tue-Thu 9-6, Fri 9-5, 1st + 3rd Sat 9-12). Print this. Stick it next to the monitor.

---

## The "top 2 of 50" rule (priority order at the start of every dial block)

Every preset queue has 30-80 records on a given morning. Kareem **does not** dial top-to-bottom alphabetically. He works in this order:

1. **Top 2 phone-tier records per preset** — Tier 1 (81-100) first, Tier 2 (61-80) next. Across 7 active presets, that's 14 records that get dialed before anything else. Highest-density yield window of the day.
2. **Sheriff Sale ≤14 days to auction** — urgency mode, regardless of tier
3. **Redemption Window ≤14 days remaining** — same logic
4. **Day 2/3 follow-ups** from yesterday's connects (the warmest records on the board)
5. **Sheriff Sale 14-30 days to auction**
6. **Lis Pendens Day 1** (first-to-market window)
7. **Probate Day 1** (less urgent but high-empathy — should not pile up)
8. **Day 7+ nurture cadence**

When Kareem is behind on a busy day, this order tells him exactly what to drop and what to keep.

## Tuesday / Wednesday / Thursday — the core 9-hour day

The day splits into **AM = FTM (courthouse data, high quality, low volume)** and **PM = Bulk Top-250 (high volume, distressed-data lists)**. AM is when prospects are home + answer rates are highest; reserve the FTM (highest-yield) records for that window.

| Time | Block | Activity |
|---|---|---|
| 9:00–9:15 | **Morning prep** | Open DataSift. Read Mike's 9:00 morning post on team group text (overnight new records, hot SMS responders, DNC flags, pull-back records). Reply with today's plan per [ISA-MIKE-COLLAB.md](ISA-MIKE-COLLAB.md). |
| 9:15–9:30 | **Top-2 prioritization** | Apply FTM priority order. Build AM dial list in Smrtphone. |
| 9:30–11:30 | **FTM Dial Block #1** (top 2 per courthouse preset, Tier 1-2) | Target: 30-40 dials. Power-dial 3-line default; drop to 1-line for probate Day 1. |
| 11:30–11:45 | **Hot handoff window** | Anyone Kareem connected with in Block #1 who scheduled with Danielle: REISimpli webform + group text on the team thread + confirmation SMS to seller. **Do this before lunch.** |
| 11:45–12:30 | **Lunch** | Step away. Phone off if possible. |
| 12:30–1:00 | **Bulk Top-250 prep** | Open today's bulk queue: which county is up today (Franklin / Montgomery / Greene rotation), which sub-preset (vacant+equity / tax-delinquent / code-violation), which week of the 3-week cycle |
| 1:00–1:05 | **Mid-day sync post** | Group text the team thread per [ISA-MIKE-COLLAB.md](ISA-MIKE-COLLAB.md): AM dial count, AM connects, anything to deconflict with Mike for PM |
| 1:05–3:30 | **Bulk Top-250 Dial Block #2** | Target: 40-55 dials. Power-dial 3-line. Lighter-touch script (the Bulk opener, not the FTM opener — see [ISA-CADENCE.md](ISA-CADENCE.md)). |
| 3:30–4:00 | **Recycle queue review** | FTM callbacks due today + bulk Tier 1 missed-connect retries. Build Block #3 list. |
| 4:00–5:00 | **Dial Block #3** (FTM follow-ups + Bulk afternoon push) | Target: 20-30 dials. People answer better afternoon than morning. Mix FTM Day-2/3 records with Bulk repeat attempts. |
| 5:00–5:30 | **DataSift hygiene + REISimpli handoffs** | Tag updates (FTM + bulk), status updates, callback scheduling, REISimpli webform submissions for any pending qualified leads |
| 5:30–6:00 | **End-of-day group text + DataSift Activity note** | EOD post on team thread per template below. Mike confirms with his EOD. Aaron reads. |

**Cruise-pace target on a 9-hour day:** 100-120 total dials (35-45 FTM + 55-75 bulk), 25-40 connects, **target 2 set appointments (= 10/week, KPI floor)**.

**Cruise target on Monday:** 60-75 dials (mix of FTM + bulk), 15-25 connects, 1-2 set appointments.

## 1st + 3rd Saturday — 9:00 AM – 12:00 PM ET (3-hour day)

Saturday is a weekend-coverage day. Many distress homeowners are home, and competition is lower (most predator dialers don't run weekends).

**This is the highest-connection-rate day of the week.** Treat it that way.

| Time | Block | Activity |
|---|---|---|
| 9:00–9:15 | **Saturday prep** | Pull Friday/Saturday-overnight new records + recent SMS responses |
| 9:15–11:30 | **Dial Block #1** (high-priority records only — Tier 1, urgent SS, urgent RW) | Target: 30-40 dials |
| 11:30–11:45 | **Hot handoff window** | If Danielle is on call rotation, ping her. Otherwise, schedule for Monday and confirmation SMS the seller. |
| 11:45–12:00 | **DataSift hygiene + Saturday end-of-day note** | Group text on the team thread — brief. See you Monday. |

**Cruise target on Saturday:** 30-40 dials, 10-15 connects, 1-2 handoffs.

## What every dial block requires Kareem to do

Before each dial block:

1. Verify Smrtphone is logged in and integrated with REISift
2. Confirm DataSift is open with today's filtered queue
3. Have [ISA-CALL-SCRIPTS.md](ISA-CALL-SCRIPTS.md) open in another window/tab — by preset section
4. Have voicemail-drop pre-recordings selected for each preset
5. Water + a notepad for verbatim quotes during connects

During each dial block:

- Tag every dial in DataSift (`dial_day_N`, `voicemail_left`, `connected`)
- After each connect: 30-60 seconds of post-call notes in DataSift Notes field
- After each qualified connect: don't wait — submit REISimpli webform within 15 minutes

After each dial block:

- 5-min hygiene pass: status updates, callback scheduling, distress-state tags

## What "good" looks like end-of-week (steady-state, post-Week-4)

- 450-550 dials/week (FTM + Bulk combined)
- 110-180 connects/week (25-35% connect rate)
- 35-60 UMBC captures/week
- **7-10 set appointments to Danielle/week** ← the KPI gate
- Pace toward 4 signed contracts/month (Aaron tracks via Danielle's REISimpli pipeline)
- 0 hard flags
- 0 records tagged Dead
- 100% of handoffs have complete REISimpli notes + team-thread group text + seller confirmation SMS
- All 3 daily Mike+Kareem touchpoints executed (morning post, mid-day sync, EOD recap)

---

## What "bad week" indicators to flag to Aaron

If any of these hit, text Aaron on the team thread same-day — don't wait for Friday retro:

- Connect rate <20% for 2+ days (phone tier issue or call timing issue)
- Zero handoffs for 3+ consecutive days (script + framework issue)
- Any hard flag (price quote, "other offers" said, missed Permission-to-say-no)
- Any TCPA-adjacent situation (unclear DNC, prospect threatening legal)
- Repeated Smrtphone or REISimpli technical failure
- Repeated tag confusion that's slowing dial pace
